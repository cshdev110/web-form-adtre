Introduction

The HTML form helps us with allowing a user to write certain information into input fields. This information can be optional or required, and they are collected and used for specific purposes creating interaction from a website. For this work, a dynamic form for a travel agency, Global Adventure, has been built to permit the company to offer tailored travel experiences for its customers.

The presented form dynamically leads a user through the fields showing how and what is required. Every input will alert the user if the data is valid or invalid via a green or red colour plush tick or cross icon. Also, some fields remove inputs that are not valid while a user types. Interaction throughout the form will be added or removed depending on user choices. The page has been developed using only HTML, CSS, and JavaScript, as required. LocalStorage is a great feature to allow a user not to lose information already typed, but far more code out of the requirements is needed.

The next section of the report depicts the issue that does not allow running this page without using HTTP or HTTPS.

Launch the site

Due to the declaration of JavaScript in the HTML index with the argument type "module" ( <script type=module></script>) to allow the use of import, the browser is blocking the request when importing, making the page unable to run correctly. That is because it does not come from an HTTP or HTTPS request and entails security implications.  CVE-2019-11730
"A vulnerability exists where if a user opens a locally saved HTML file, this file can use file: URIs to access other files in the same directory or sub-directories if the names are known or guessed. The Fetch API can then be used to read the contents of any files stored in these directories and they may uploaded to a server." (Bug 1558299: Luigi Gubello, CVE-2019-11730, 2019, https://cve.mitre.org/cgi-bin/cvename.cgi?name=CVE-2019-11730)

This issue can be carried through successfully and straightforward with two simple approaches: First of all, apologies to the examiner for this point in the assignment.

  • First approach: Considering this issue might be burdened, and so, to make it easy, this site is deployed in my github repository at the following URL:
    https://cshdev110.github.io/web-form-adtre/. It is reachable and will be on until it is no more required.
  • Second approach: Run a local server. There are many ways to do so, although running a Python or php server is the easiest. Therefore, having Python or php install, open a terminal located where the index.html file is, and run one of these three options:
    • Phyton2:
      $ python -m SimpleHTTPServer 8000
    • Phyton3:
      $ python -m http.server 8000
    • php:
      $ php -S localhost:8000
    • Lastly, open a browser and type the URL: http://localhost:8000

      • Exmaple

      The page will be then launched, and the form will work correctly.

Structure

As shown in the picture, the site is arranged using five primary folders and one sub-folder (report folder inside the img folder to keep the image for this report). The outsider folder has the three leading files that govern the entire site; index.html, index.js, and style.css.

  • The "css" folder: has three CSS files that style individual parts of the form page and one report.css, which styles the report page fully. Each specific CSS file name means where they are styling: style-footer, the footer, style-header, the header, style-main, the main section, and the report.css the report page.
  • The "icons" folder: Contains all the icons used on the page, including in the report.
  • The "img" folder: contains the site's images and the report's images in a report subfolder.
  • The "js" folder: The first file, report.js, has the code to manage the report page, and the vars_html_style.js are the exported variables used to handle the dynamics to add HTML elements as needed.
  • The "report" folder: has the report.html file, which is this current report page.
